# AirSense Codebase Review Notes

This repository review was generated by automated inspection and test execution.

## Key findings snapshot

1. `get_time_series` uses a free-form `station` query string to build a file path and reads CSV on every request.
2. `/predict/map-global` and `/metrics/projections` build large response lists row-by-row; this is expensive at runtime.
3. Training scripts use random train/test splits for time-dependent data, introducing leakage risk.
4. Clustering training can fail when KMeans candidate scores are invalid (`best_km` can remain `None`).
5. API security defaults are permissive (open CORS, no upload size/type guardrails).
6. Test suite validates major paths but misses negative/security/performance cases.
